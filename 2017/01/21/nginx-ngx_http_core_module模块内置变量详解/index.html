nginx ngx_http_core_module模块内置变量详解

发表于 2017-01-22 | 分类于 nginx
了解ngx_http_core_module模块nginx内置变量。

ngx_http_core_module支持名字匹配Apache Server变量的内置变量,首先这些变量体现了客户端请求头 字段,比如$http_user_agent, $http_cookie,等等,而且还有以下的其他变量:

##请求中的的参数名，即“?”后面的arg_name=arg_value形式的arg_name
$arg_name 

##请求中的参数值，即“?”后面的arg_name=arg_value形式的arg_value
$args

##客户端地址的二进制形式,对于IPV4固定长度为4个字节，对于IPV6固定长度为16个字节
$binary_remote_addr

##传输给客户端的字节数,响应头不计算在内,这个变量等同于与Apache的mod_log_config模块的'%B'
$body_bytes_sent

##传输给客户端的字节数
$bytes_sent

##TCP连接的序列号
$connection

##TCP连接当前的请求数量
$connection_requests

##"Content-Length"请求头字段
$content_length

##"content_type"请求头字段
$content_type

##cookie名
$cookie_name

##当前请求的文档根目录或别名
$document_root

##与$uri一样
$document_uri

##优先级如下：HTTP请求行的主机名>”HOST”请求头字段>符合请求的服务器名
$host

##主机名
$hostname

##匹配任意请求头字段； 变量名中的后半部分“name”可以替换成任意请求头字段，如在配置文件中需要
##获取http请求头：“Accept-Language”，那么将“－”替换为下划线，大写字母替换为小写，形如：
##$http_accept_language即可
$http_name

##如果开启了SSL安全模式，值为“on”，否则为空字符串
$https

##如果请求中有参数，值为“?”，否则为空字符串
$is_args

##用于设置响应的速度限制，详见 limit_rate
$limit_rate

##当前的Unix时间戳
$msec

##nginx版本
$nginx_version

##工作进程的PID
$pid

##如果请求来自管道通信,值为"p",否则为"."
$pipe

##获取代理访问服务器的客户端地址，如果是直接访问，该值为空字符串
$proxy_protocol_addr

##与$args一样
$query_string

##当前请求的文档根目录或别名的真实路径，会将所有符号连接转换为真实路径。
$realpath_root

##客户端地址
$remote_addr

##客户端端口
$remote_port

##用于HTTP基础认证服务的用户名
$remote_user

##代表客户端的请求地址
$request

##客户端的请求主体
##此变量可在location中使用,将请求主体通过proxy_pass,fastcgi_pass,uwsgi_pass, 和 
##scgi_pass传递给下一级的代理服务器
$request_body

##将客户端请求主体保存在临时文件中。文件处理结束后，此文件需删除。如果需要之一开启此功能，需
##要设置client_body_in_file_only。如果将次文件传递给后端的代理服务器，需要禁用request 
##body，即设置proxy_pass_request_body off，fastcgi_pass_request_body off, 
##uwsgi_pass_request_body off, or scgi_pass_request_body off 
$request_body_file

##如果请求成功,值为”OK”，如果请求未完成或者请求不是一个范围请求的最后一部分，则为空
$request_completion

##当前连接请求的文件路径，由root或alias指令与URI请求生成
$request_filename

##由16字节的随机数生成的独一无二的16进制的请求标识
$request_id

##请求的长度(包括请求的地址, http请求头和请求主体)
$request_length

##请求的方法,通常是"GET"或者"POST"
$request_method

##处理客户端请求使用的时间,从读取客户端的第一个字节开始计时。
$request_time

##完整的原始请求URI(带参数)
$request_uri

##请求的协议,"http"或者"https"
$scheme

##可以设置任意http响应头字段； 变量名中的后半部分“name”可以替换成任意响应头字段，如需要设置
##响应头Content-length，那么将“－”替换为下划线，大写字母替换为小写，形如：
##$sent_http_content_length 4096即可。
$sent_http_name

##接受一个请求的服务端地址
$server_addr

##接受一个请求的服务端名字
$server_name

##接受一个请求的服务端端口
$server_port

##请求协议,通常是"HTTP/1.0","HTTP/1.1"，或者"HTTP/2.0"
$server_protocol

##响应状态码
$status

##客户端TCP连接信息,支持TCP_INFO socket选项
 $tcpinfo_rtt, $tcpinfo_rttvar, $tcpinfo_snd_cwnd, $tcpinfo_rcv_space 

##ISO 8601标准格式的本地时间
$time_iso8601

##普通的日志格式的本地时间
$time_local

##当前请求的URI,$uri的值可能在请求的过程中发生改变,例如:当发生一个内部的重定向或者当使用索引文件的时候
$uri
