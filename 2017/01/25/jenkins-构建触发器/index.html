jenkins 构建触发器

发表于 2017-01-26 | 分类于 jenkins
让项目构建更加自动化.

Build whenever a SNAPSHOT dependency is built

如果选中这个选项,jenkins将会解析此项目的POM文件,并看看它的快照所依赖的所有东西是否也被构建,
为了当依赖项目被构建的时候一个新的SNAPSHOT的jar包会被构建,jenkins会建立依赖关系,jenkins会
调度此项目的构建.    
这个对持续集成是十分方便的,jenkins会从POM文件中的<dependency>,<plugin>s,<extension>s要
素中检查依赖
如果这个选项有问题,那么就去掉勾选.

触发远程构建

如果你想通过访问一个特殊的预定义的URL来触发项目的构建则启动此项(对于写脚本是十分方便的)，通过
预定义的口令,根据特定形式的URL来触发新的构建.

Build after other projects are built

为了当其他的项目构建完成之后,对于这个项目一个新的构建被列入计划之中,这对于完成一个构建之后运
行大量的测试是十分便利的,例如:这个配置完善了在"Post-build Actions"中的"Build other 
projects"对于上游项目的构建方案,因为这个选项对于下游项目的构建是最好的选择.

Build periodically

提供了一个像cron的定时任务一样的功能来定时构建.

Build when a change is pushed to GitLab

当GitLab上面代码发生改变的时候触发一个新的构建.

Poll SCM

配置jenkins轮训SCM的变化,如果有变化就触发构建,轮训的周期和cron一样.
注意此项对于SCM是一个巨大的消耗,因为服务器会扫描整个工作空间并校验.
